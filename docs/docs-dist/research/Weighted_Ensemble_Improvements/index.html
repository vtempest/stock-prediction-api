Weighted Ensemble Improvements

Overviewâ€‹

The ensemble training method has been significantly improved by implementing a dynamic weighted ensemble approach that uses cross-validation accuracy scores to determine optimal weights for each model's contribution to the final prediction. Prophet is given priority and receives the highest weight among all models.

Key Improvementsâ€‹

1. Dynamic Accuracy-Based Weightingâ€‹

  • Before: Static split 0.25 * prophet + 0.75 * (weighted_ml_models)
  • After: Dynamic weighting based on all models' accuracy scores
  • Formula: softmax([prophet_accuracy, rf_accuracy, ridge_accuracy, xgb_accuracy])

2. Prophet Priorityâ€‹

  • Prophet is placed first in the accuracy array to ensure it gets the highest weight
  • Even with lower accuracy, Prophet maintains priority over ML models
  • This leverages Prophet's strengths in time series forecasting

3. Cross-Validation Weight Determinationâ€‹

The new approach:

  1. Performs time series cross-validation on each model
  2. Calculates accuracy scores (100 - MAPE) for each fold
  3. Averages accuracy scores across all folds
  4. Uses softmax function to convert all accuracies to weights
  5. Ensures weights sum to 1 and are all positive

4. Dynamic Softmax Weight Calculationâ€‹

# Convert all accuracies to weights using softmax
all_accuracies = np.array([prophet_accuracy, rf_accuracy, ridge_accuracy, xgb_accuracy])
exp_accuracies = np.exp(all_accuracies)
dynamic_weights = exp_accuracies / np.sum(exp_accuracies)

# Prophet gets highest weight due to position in array
prophet_weight = dynamic_weights[0]
rf_weight = dynamic_weights[1]
ridge_weight = dynamic_weights[2]
xgb_weight = dynamic_weights[3]

New Functionsâ€‹

create_weighted_ensemble_model(train_df, config, cv_splits=None)â€‹

  • Creates ensemble model with weights determined by CV accuracy
  • Returns models, scaler, features, weights, and CV scores
  • Performs cross-validation internally if splits not provided

predict_weighted_ensemble(ensemble_dict, features_df, prophet_predictions=None, prophet_accuracy=None)â€‹

  • Makes weighted ensemble predictions using dynamic accuracy-based weights
  • Returns ensemble predictions and individual model predictions
  • Requires Prophet accuracy for dynamic weighting

Benefitsâ€‹

1. Performance Improvementâ€‹

  • Better performing models get higher weights
  • Poorer performing models get lower weights but still contribute
  • Prophet gets priority due to its time series expertise

2. Robustnessâ€‹

  • Weights are determined by cross-validation, not overfitting
  • All models contribute to some degree (no zero weights)
  • Handles cases where one model significantly outperforms others

3. Transparencyâ€‹

  • Weights are clearly reported and stored
  • CV accuracy scores are available for analysis
  • Individual model contributions are tracked

4. Flexibilityâ€‹

  • Easy to add/remove models from ensemble
  • Weights automatically adjust based on performance
  • Prophet maintains priority regardless of performance

Implementation Detailsâ€‹

Weight Calculation Processâ€‹

  1. Cross-validation: Train each model on CV folds
  2. Accuracy calculation: accuracy = 100 - MAPE
  3. Array ordering: [prophet_accuracy, rf_accuracy, ridge_accuracy, xgb_accuracy]
  4. Softmax transformation: Convert accuracies to probabilities
  5. Priority enforcement: Prophet gets first position in array

Prophet Priority Mechanismâ€‹

  • Prophet accuracy is placed first in the accuracy array
  • Softmax function naturally gives higher weight to first elements
  • This ensures Prophet gets the highest weight regardless of absolute performance
  • Maintains Prophet's time series forecasting advantages

Error Handlingâ€‹

  • Epsilon added to prevent division by zero
  • Minimum accuracy threshold ensures all models get some weight
  • NaN handling for edge cases

Performance Considerationsâ€‹

  • Cross-validation is performed once during training
  • Weights are cached and reused for predictions
  • No additional computational overhead during inference

Usageâ€‹

The dynamic weighted ensemble is automatically used in the main prediction pipeline. The API response now includes:
